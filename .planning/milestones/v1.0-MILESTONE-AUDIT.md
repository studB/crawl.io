# Milestone v1.0 — Audit Report

**Audited:** 2026-04-18
**Status:** `tech_debt` — all requirements satisfied, 3 non-blocking debt items tracked

## Summary

crawl.io v1.0 delivers the full value promised in PROJECT.md: a TypeScript CLI that parses a markdown config file, crawls the described page via Playwright (CSS + XPath + nested iframes), logs into Naver Cafe when env-var credentials are present, persists the session across runs, falls back to a headed browser for captcha/2FA, and appends results back into the same markdown file under `# Output`. The tool is npm-publishable via `crawl run <file.md>`.

All 29 v1 REQ-IDs are delivered with traceable code evidence. Cross-phase wiring is clean — `parseConfigFile → runCrawl → writeOutputToFile` flows end-to-end, Phase 3's `ensureAuthenticated` plugs in between context creation and `page.goto`, Phase 4's CLI never reaches into internal auth modules. No integration gaps detected.

## Cross-Phase Integration: all green

- **Phase 1 → Phase 2:** `parseConfigFile` imported by `runCrawl`; `ConfigParseError` caught and mapped to `CrawlResult.error.code='config_parse'` before browser launch.
- **Phase 2 → Phase 4:** CLI `run` handler imports `runCrawl` from the public barrel; no library reach-through.
- **Phase 3 → Phase 2:** `ensureAuthenticated` runs between `launchBrowser` + context creation and `page.goto(url)`. Identity-swap + handle rebind handled cleanly after headed fallback.
- **Phase 4 → Phase 3:** CLI never imports from `src/auth/` — auth is fully internal. Verified by grep.
- **CrawlErrorCode union:** 10 literal members including Phase 3's 3 new codes (`auth_missing_credentials`, `auth_failed`, `captcha_unresolved`).

## End-to-End Flows: 7/7 ✓

1. Public URL crawl happy path — parseConfigFile → launchBrowser → goto → waitFor → extract → writeOutput → exit 0.
2. Naver authed crawl — `.crawl-session.json` written after successful login.
3. Session reuse — second run skips login via `sessionLooksValid()` gate.
4. Captcha headed fallback — close headless → relaunch headed → poll → save session → relaunch headless → continue crawl.
5. Missing config — CLI emits scrubbed error to stderr, exits 1 before calling `runCrawl`.
6. `--help` — both top-level and `run --help` include env vars + exit codes.
7. Publishable tarball — `npm pack --dry-run` + `npm publish --dry-run` both exit 0; shebang preserved in `dist/bin/crawl.js`.

### Phase 4 — npm Tarball Bloat + Placeholder URLs

- 72 compiled `.test.*` files ship inside the npm tarball (~117 kB). Not a correctness issue but noisy for consumers. Fix via `tsconfig.json` `exclude: ["**/*.test.ts", "**/*.integration.test.ts"]` OR an `.npmignore` OR a narrower `files` pattern.
- `repository.url`, `bugs.url`, `homepage` were removed from `package.json` per the code-review fix (WR-03) — they had `TBD` placeholders. User must add real URLs before running actual `npm publish`. `npm publish --dry-run` passes without them.
