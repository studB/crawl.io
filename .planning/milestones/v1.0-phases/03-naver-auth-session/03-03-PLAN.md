       Required exports:

       Returns the (possibly replaced) `Page` the caller should use for subsequent `goto`. In the no-auth and reuse-succeeds paths, returns the SAME page. In the headed-fallback path, closes the headless browser and returns a fresh page from a new headless browser — see flow below.

       Implementation rules (exact):

       ```typescript
       import type { Browser, BrowserContext, Page } from 'playwright';
       import { chromium } from 'playwright';

       import { CrawlError } from '../crawler/errors';
       import { scrubPaths } from '../crawler/output';
       import {
         classifyPostLogin,
         CAPTCHA_SELECTORS,
         hasNaverSessionCookies,
         isNaverHost,
         type CookieLike,
       } from './detect';
       import {
         HEADED_POLL_INTERVAL_MS,
         pollUntilLoggedIn,
         resolveHeadedTimeoutMs,
       } from './headed';
       import {
         NAVER_LOGIN_URL,
         readNaverCredentials,
         submitNaverLoginForm,
       } from './naver';
       import { sessionExists, sessionFilePath } from './session';

       export interface AuthContextOptions {
         /** Override for tests; default `process.env`. */
         readonly env?: NodeJS.ProcessEnv;
         /** Override for tests; default `process.cwd()`. */
         readonly cwd?: string;
         /** Override for tests; default is `chromium.launch`. */
         readonly launchHeadless?: (storageStatePath?: string) => Promise<{ browser: Browser; context: BrowserContext; page: Page }>;
         readonly launchHeaded?: (storageStatePath?: string) => Promise<{ browser: Browser; context: BrowserContext; page: Page }>;
         /** Override for tests; default writes stderr. */
         readonly onHeadedOpened?: (timeoutMs: number) => void;
       }

       /**
        * Probe each CAPTCHA_SELECTORS entry for presence on a Page within a
        * tight internal budget. Never throws — failures are reported as false.
        */
       async function probeCaptchaSelectors(page: Page): Promise<boolean[]> {
         const hits: boolean[] = [];
         for (const sel of CAPTCHA_SELECTORS) {
           try {
             // count() is cheap and non-throwing for non-matching selectors.
             const count = await page.locator(sel).count();
             hits.push(count > 0);
           } catch {
             hits.push(false);
           }
         }
         return hits;
       }

       async function readCookies(context: BrowserContext): Promise<CookieLike[]> {
         const raw = await context.cookies();
         return raw.map((c) => ({ name: c.name, value: c.value }));
       }

       export async function ensureAuthenticated(
         page: Page,
         targetUrl: string,
         browser: Browser,
         contextOpts?: AuthContextOptions,
       ): Promise<Page> {
         const env = contextOpts?.env ?? process.env;
         const cwd = contextOpts?.cwd ?? process.cwd();

         // Fast path: not a Naver host → caller's session reuse is still in effect
         // (launchBrowser already loaded .crawl-session.json if present), but we
         // do NOT attempt a Naver login for non-Naver URLs.
         if (!isNaverHost(targetUrl)) {
           return page;
         }

         // Naver host. Credentials may or may not be set:
         //   - Both set → we can attempt login if the existing session is stale.
         //   - Unset → we proceed WITHOUT attempting a login; the target-page
         //     probe decides whether to emit auth_missing_credentials.
         const context = page.context();

         // If an existing session already carries NID_AUT + NID_SES, trust it
         // until the target page proves otherwise (D4 + D3-CONTEXT §Session reuse).
         const preCookies = await readCookies(context);
         if (hasNaverSessionCookies(preCookies)) {
           return page;
         }

         // Session did not carry the cookies. Try a login if we have creds;
         // otherwise we defer to the runner's post-goto redirect detection
         // (if the goto redirects to nid.naver.com, the runner's next classify
         // round will surface auth_missing_credentials).
         const idPresent = env.NAVER_ID !== undefined && env.NAVER_ID.length > 0;
         const pwPresent = env.NAVER_PW !== undefined && env.NAVER_PW.length > 0;
         if (!idPresent || !pwPresent) {
           if (await sessionExists(cwd)) {
             // Existing (stale) session + no creds → let the runner see the
             // redirect and surface auth_missing_credentials.
             return page;
           }
           throw new CrawlError(
             'auth_missing_credentials',
             'NAVER_ID and NAVER_PW must both be set to log into ' + scrubPaths(targetUrl),
           );
         }

         // Attempt headless login.
         const creds = readNaverCredentials(env);
         await submitNaverLoginForm(page, creds);

         // Classify post-submit state.
         const postUrl = page.url();
         const postCookies = await readCookies(context);
         const selectorHits = await probeCaptchaSelectors(page);
         const cls = classifyPostLogin({
           currentUrl: postUrl,
           cookies: postCookies,
           captchaSelectorHits: selectorHits,
         });

         if (cls === 'logged_in') {
           // Persist the fresh session to disk for next-run reuse.
           await context.storageState({ path: sessionFilePath(cwd) });
           return page;
         }

         if (cls === 'captcha') {
           return await runHeadedFallback(browser, targetUrl, contextOpts);
         }

         // login_required / unknown after a submit → credentials were wrong or
         // the form changed; classify as auth_failed.
         throw new CrawlError(
           'auth_failed',
           'login did not produce NID_AUT/NID_SES cookies (classification=' + cls + ')',
         );
       }

       async function runHeadedFallback(
         origBrowser: Browser,
         targetUrl: string,
         opts?: AuthContextOptions,
       ): Promise<Page> {
         const env = opts?.env ?? process.env;
         const cwd = opts?.cwd ?? process.cwd();
         const timeoutMs = resolveHeadedTimeoutMs(env);
         const sessionPath = sessionFilePath(cwd);
         const onOpened = opts?.onHeadedOpened ?? ((ms: number) => {
           process.stderr.write(
             '\u26A0 Captcha/2FA detected — resolve it in the visible browser window. ' +
             'Waiting up to ' + Math.floor(ms / 1000) + 's...\n',
           );
         });

         // Close headless. Best-effort — we never want close errors to shadow
         // the auth flow.
         try { await origBrowser.close(); } catch { /* swallow */ }

         const launchHeaded = opts?.launchHeaded ?? (async (storage?: string) => {
           const b = await chromium.launch({ headless: false });
           const ctxInit: Parameters<Browser['newContext']>[0] = {};
           if (storage !== undefined) ctxInit.storageState = storage;
           const ctx = await b.newContext(ctxInit);
           const pg = await ctx.newPage();
           return { browser: b, context: ctx, page: pg };
         });

         const headed = await launchHeaded(await (async () => {
           // Reuse the on-disk session if one exists; otherwise launch fresh.
           return (await sessionExists(cwd)) ? sessionPath : undefined;
         })());

         try {
           onOpened(timeoutMs);
           await headed.page.goto(NAVER_LOGIN_URL);

           await pollUntilLoggedIn({
             isLoggedIn: async () => {
               const cookies = await readCookies(headed.context);
               if (hasNaverSessionCookies(cookies)) return true;
               try {
                 const u = new URL(headed.page.url());
                 // "Navigated back to naver main" — a generous additional signal.
                 if (u.host.endsWith('naver.com') && !u.host.startsWith('nid.')) return true;
               } catch { /* ignore */ }
               return false;
             },
             timeoutMs,
             intervalMs: HEADED_POLL_INTERVAL_MS,
           });

           // Persist fresh session before tearing down the headed browser.
           await headed.context.storageState({ path: sessionPath });
         } finally {
           try { await headed.page.close(); } catch { /* swallow */ }
           try { await headed.context.close(); } catch { /* swallow */ }
           try { await headed.browser.close(); } catch { /* swallow */ }
         }

         // Relaunch HEADLESS with the now-fresh session (file is source of truth — see 03-CONTEXT.md §specifics).
         const launchHeadless = opts?.launchHeadless ?? (async (storage?: string) => {
           const b = await chromium.launch({ headless: true });
           const ctxInit: Parameters<Browser['newContext']>[0] = {};
           if (storage !== undefined) ctxInit.storageState = storage;
           const ctx = await b.newContext(ctxInit);
           const pg = await ctx.newPage();
           return { browser: b, context: ctx, page: pg };
         });
         const fresh = await launchHeadless(sessionPath);
         // The caller's runner holds references to the prior handle; returning
         // only the new Page is insufficient. We attach the fresh browser/context
         // to the returned page so runner can close them in `finally`.
         // See runner edit in Task 2 — it uses page.context().browser() to reach
         // them, which is a stable Playwright API.
         // eslint-disable-next-line no-void — explicit tie-in with targetUrl for future logging
         void targetUrl;
         return fresh.page;
       }
       ```

       Notes on the contract:
       - `ensureAuthenticated` returns the Page the caller should continue with. In the non-headed path it's the SAME page. In the headed path it's a page from a FRESH headless browser (the original one was closed). The caller (runner) must handle replacement — covered in Task 2.
       - `CrawlError` with `auth_missing_credentials` / `auth_failed` / `captcha_unresolved` (the last comes from `pollUntilLoggedIn` → propagates unchanged) flows into the runner's existing catch block unchanged.
       - Error-message path safety: `scrubPaths(targetUrl)` in the missing-creds branch is the only place this module interpolates a path-like string; the other branches interpolate only classification names and cookie booleans.
