<objective>
Deliver the three Playwright-bound crawler modules — `browser.ts` (Chromium lifecycle), `frame.ts` (nested-iframe descent), `extract.ts` (CSS + XPath extraction with timeout handling) — plus an integration test suite exercising them against a local three-level HTML fixture served via `file://` URLs. A local fixture is explicitly permitted for this plan per 02-CONTEXT.md: "PROJECT.md's no-fixtures rule applies to Naver E2E tests, not the iframe descent mechanism."

Purpose: These three modules are where all the real crawling work happens. Plan 04 composes them into `runCrawl`; none of them talks to the filesystem output layer (Plan 02's output.ts). Keeping these Playwright-bound pieces in a single plan (rather than splitting browser / frame / extract each into its own plan) is justified by their tight coupling — they share the Playwright Page/FrameLocator type graph, and splitting would force a cascade of unit-level mocks that provide no confidence the real browser works.

Output: Three source modules, one unit test (frame.test.ts — pure string logic for the path-builder helper), two test files hitting Playwright (extract.test.ts for error-mapping behavior on a minimal page, extract.integration.test.ts for the two-level iframe + CSS vs XPath cross-check), three HTML fixtures, a vitest config with a 60s timeout for `*.integration.test.ts`.
</objective>

    ```ts
    import type { Page } from 'playwright';
    import type { SelectorSpec } from '../config/types';
    import type { FrameTarget } from './frame';
    import { descendToFrame } from './frame';
    import { CrawlError } from './errors';

    /**
     * Wait for the `waitFor` selector to appear on the top-level page, with timeout.
     * Maps Playwright's TimeoutError to CrawlError('timeout', detail).
     * The detail string MUST include BOTH the selector and the timeout value per CONTEXT.md.
     */
    export async function waitForReady(
      page: Page,
      waitFor: string | undefined,
      timeout: number,
    ): Promise<void>;

    /**
     * Resolve every selector against its declared frame (or the top page if frame is absent).
     * Returns a { fieldName: textContent } map.
     * Per-field failures are attributed — error detail names the offending field.
     * This function is the SOLE Phase-2 throw site for CrawlError('frame_not_found', ...).
     */
    export async function extractFields(
      page: Page,
      selectors: Record<string, SelectorSpec>,
    ): Promise<Record<string, string>>;

    /**
     * Compose the Playwright selector string for a given SelectorSpec.
     * CSS: returns the selector as-is. XPath: prefixes with `xpath=`.
     * Pure function — exported for unit testing.
     */
    export function toPlaywrightSelector(spec: SelectorSpec): string;
    ```

    Implementation:

    - `toPlaywrightSelector(spec)`:
      - `if (spec.engine === 'xpath') return 'xpath=' + spec.selector;`
      - `return spec.selector;`
      - Rationale: Playwright's `page.locator(sel)` defaults to CSS; the `xpath=` engine prefix switches to XPath. This is the documented Playwright API and the simplest path to CRWL-04 + CRWL-05.

    - `waitForReady(page, waitFor, timeout)`:
      - If `waitFor === undefined`: return immediately (no-op — `timeout` still applies to `page.goto` upstream in Plan 04's runner).
      - Else: wrap `page.waitForSelector(waitFor, { timeout })` in try/catch. On catch:
        ```ts
        if (err instanceof Error && err.name === 'TimeoutError') {
          throw new CrawlError('timeout',
            'waitFor selector `' + waitFor + '` did not appear within ' + timeout + 'ms');
        }
        throw new CrawlError('unknown', (err as Error).message);
        ```
      - Catch by `err.name === 'TimeoutError'` rather than `err instanceof errors.TimeoutError` — the name check survives Playwright version bumps and ESM/CJS interop quirks.

    - `extractFields(page, selectors)`:
      - Iterate `Object.entries(selectors)`. For each `[name, spec]`:
        1. `const target: FrameTarget = descendToFrame(page, spec.frame);`
        2. `const pwSelector = toPlaywrightSelector(spec);`
        3. Wrap in try/catch:
           ```ts
           try {
             const locator = target.locator(pwSelector);
             const text = await locator.first().textContent({ timeout: 5000 });
             if (text === null) {
               throw new CrawlError('selector_miss',
                 'selector `' + spec.selector + '` for field `' + name + '` returned null');
             }
             out[name] = text.trim();
           } catch (err) {
             if (err instanceof CrawlError) throw err;
             const e = err as Error;
             if (e.name === 'TimeoutError') {
               // Could be: frame not found at descent, OR element not present.
               // Distinguish by whether frame is declared.
               if (spec.frame && spec.frame.length > 0) {
                 throw new CrawlError('frame_not_found',
                   'frame path ' + JSON.stringify(spec.frame) + ' not reachable for field `' + name + '`');
               }
               throw new CrawlError('selector_miss',
                 'selector `' + spec.selector + '` for field `' + name + '` did not match within 5000ms');
             }
             throw new CrawlError('extraction_failed',
               'failed to extract `' + name + '`: ' + e.message);
           }
           ```
      - Return `out`.
      - The 5000ms intra-field timeout is separate from `rules.timeout` by design: `rules.timeout` is the page-load / waitFor budget (big), the per-field extraction is expected to be sub-second on a rendered page. Plan 04's runner passes `rules.timeout` to `page.goto` and `waitForReady`; extraction uses a tight internal default. This matches Playwright's guidance — per-element ops should not share the page-load budget.
