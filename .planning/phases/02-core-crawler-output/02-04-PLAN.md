<objective>
Assemble the public `runCrawl(configPath)` entrypoint by wiring Phase 1's parser, Plan 02-01's error types, Plan 02-02's markdown writeback, and Plan 02-03's browser/frame/extract modules into a single orchestrator that:

1. Parses the config file (catches `ConfigParseError` → writes `config_parse` error entry).
2. Launches Chromium, navigates, waits for the ready selector, extracts every declared field, closes the browser in a `finally`.
3. Renders a `CrawlResult` and appends it to the config file's `# Output` section — ALWAYS, on success or failure.
4. Returns the `CrawlResult` without calling `process.exit` (caller owns exit semantics — Phase 4 CLI will map `status === 'error'` to exit 1).

Purpose: This plan is where every Phase 2 requirement converges (the extraction + envelope + writeback chain). Integration tests use the Plan 03 fixture chain plus one-off configs written to `os.tmpdir()` (so we can mutate the file and read it back without polluting the repo).

**OUT-05 split note:** OUT-05 ("process exits with non-zero code on any failure") is split across phases by explicit CONTEXT.md decision. Phase 2 (this plan) delivers the error envelope: `CrawlResult.status === 'error'` with a populated `error: { code, message, stack? }`, plus the written error entry in `# Output`. Phase 4 (CLI) will consume that envelope and map `status === 'error'` to `process.exit(1)` inside the CLI wrapper. That is why OUT-05 is NOT in this plan's `requirements` frontmatter — the traceability ledger is updated by the verifier after execution, not by the planner.

Output: `src/crawler/runner.ts`, end-to-end integration tests, the crawler barrel, and updated package-root exports.

    ```ts
    import { parseConfigFile, ConfigParseError } from '../config/index';
    import type { CrawlJob } from '../config/index';
    import type { CrawlResult, CrawlErrorCode } from './types';
    import { CrawlError } from './errors';
    import { launchBrowser, closeBrowser, type BrowserHandle } from './browser';
    import { waitForReady, extractFields } from './extract';
    import { renderEntry, writeOutputToFile } from './output';

    export async function runCrawl(configPath: string): Promise<CrawlResult> {
      const startedAt = new Date();
      const startNs = process.hrtime.bigint();
      let url = '';

      // Helper: build a CrawlResult and ALWAYS write it to the output file before returning.
      const finalize = async (partial: Omit<CrawlResult, 'configPath' | 'startedAt' | 'durationMs'>): Promise<CrawlResult> => {
        const durationMs = Number((process.hrtime.bigint() - startNs) / 1_000_000n);
        const result: CrawlResult = {
          ...partial,
          configPath,
          startedAt,
          durationMs,
        };
        // Best-effort writeback. If writeback itself fails, we still return the result
        // (the caller / CLI will surface the exit code). We do NOT throw from runCrawl.
        try {
          await writeOutputToFile(configPath, renderEntry(result));
        } catch {
          // swallow — returning the result is still meaningful
        }
        return result;
      };

      // Helper: build the error payload. Spread `stack` ONLY when present so
      // exactOptionalPropertyTypes + output.ts's conditional stack-spread are both happy.
      const errorPayload = (code: CrawlErrorCode, message: string, stack?: string) => ({
        code,
        message,
        ...(stack !== undefined ? { stack } : {}),
      });

      // --- Phase 1 parse stage ---
      let job: CrawlJob;
      try {
        job = await parseConfigFile(configPath);
        url = job.url;
      } catch (err) {
        if (err instanceof ConfigParseError) {
          // ConfigParseError's `.stack` (inherited from Error) is useful for debugging;
          // include it on the envelope per CONTEXT.md `error: { code, message, stack? }`.
          return finalize({
            status: 'error',
            url: '',
            error: errorPayload('config_parse', err.issues.join('; '), err.stack),
          });
        }
        const e = err as Error;
        return finalize({
          status: 'error',
          url: '',
          error: errorPayload('unknown', e.message, e.stack),
        });
      }

      // --- Browser stage ---
      let handle: BrowserHandle | undefined;
      try {
        handle = await launchBrowser();

        // page.goto with configured timeout — maps Playwright TimeoutError to 'timeout' / network error to 'network'
        try {
          await handle.page.goto(url, { timeout: job.rules.timeout });
        } catch (err) {
          const e = err as Error;
          if (e.name === 'TimeoutError') {
            throw new CrawlError('timeout',
              'page.goto(' + url + ') did not complete within ' + job.rules.timeout + 'ms');
          }
          // Playwright wraps net::ERR_* as a generic Error with name 'Error'
          throw new CrawlError('network', 'navigation failed: ' + e.message);
        }

        // waitForReady applies the same timeout budget on the waitFor selector (CRWL-02 + CRWL-03)
        await waitForReady(handle.page, job.rules.waitFor, job.rules.timeout);

        // extractFields — already throws CrawlError on failure
        const fields = await extractFields(handle.page, job.selectors);

        return await finalize({ status: 'ok', url, fields });
      } catch (err) {
        let code: CrawlErrorCode = 'unknown';
        let message = (err as Error).message;
        // Standard Error.stack — present on every JS Error subclass including CrawlError.
        // Populate per CONTEXT.md error shape { code, message, stack? }; undefined safely
        // skips the spread in errorPayload.
        const stack: string | undefined = (err as Error).stack;
        if (err instanceof CrawlError) {
          code = err.code;
          message = err.detail ?? err.message;
        }
        return await finalize({ status: 'error', url, error: errorPayload(code, message, stack) });
      } finally {
        if (handle !== undefined) {
          await closeBrowser(handle);
        }
      }
    }
    ```

    Contract notes locked by this implementation:
    - **No `process.exit`.** 02-CONTEXT.md locks this. Grep must confirm. The caller (Phase 4 CLI) will map `status === 'error'` to `process.exit(1)` — that is the second half of OUT-05 and belongs in Phase 4.
    - **Finalize-always.** Every return path flows through `finalize`, which calls `writeOutputToFile`. On a thrown error, the `catch` block in the try/catch around the browser stage funnels into `finalize`; the parse-stage catch funnels into `finalize` directly. If the writeback itself fails, we swallow (result is still returned — caller decides what to do).
    - **Finally-close.** `closeBrowser(handle)` runs in `finally`, ensuring Chromium never leaks even if `finalize` throws.
    - **`url = ''`** on config_parse error — because we don't know the URL yet. The output entry's `meta.url` will be an empty string; acceptable, as the user already knows which file they pointed at.
    - **Duration uses `process.hrtime.bigint()`** — monotonic and not affected by wall-clock jumps. Truncated to ms via bigint division.
    - **CrawlError.detail is preferred over .message** when mapping to `error.message`, because `detail` is the user-facing explanation; `.message` is already `'[code] detail'` which would duplicate the code in the output.
    - **`stack` population.** Every error path reads the caught `Error.stack` (a standard property on every JS Error subclass, including `CrawlError` via inheritance). It is passed through `errorPayload`, which spreads `stack` into the envelope's `error` object ONLY when defined. When the caught value has no stack (e.g., a non-Error thrown), `err.stack` is `undefined` and no `stack` key appears in the output — matching the 02-02 renderEntry contract (Test 6 negative, Test 9 positive).
